MDX is Lisible’s default article format and the format of every page in this documentation. It keeps Markdown and lets prose import components. Reserve interactions for cases that genuinely improve understanding; a paragraph or table remains preferable for static information.

## Spoiler

The component hides an answer until explicit activation: <Spoiler>the content remains in HTML and must therefore stay non-sensitive.</Spoiler>

:::warning[Not for secrets]
A spoiler is a presentation effect, not access control. Never put a key, private answer or confidential data inside it.
:::

## Hydration rules

- `client:load` for an immediately necessary interaction;
- `client:idle` for a non-critical enhancement;
- `client:visible` for a component far below the fold;
- no client directive for a static Astro component.
